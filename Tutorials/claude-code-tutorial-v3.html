快速安裝

保留必要資訊，不再展開長版安裝精靈
1. 檢查環境

支援 macOS 13+、Windows 10 Build 1809+、Ubuntu 20.04+。Windows 必須先安裝 Git for Windows。

2. 執行安裝

macOS / Linux：curl -fsSL https://claude.ai/install.sh | bash
PowerShell：irm https://claude.ai/install.ps1 | iex

3. 驗證版本

安裝後執行 claude --version，再跑 claude doctor 做完整檢查。

4. 首次登入

第一次啟動 claude 會開啟瀏覽器登入；若未自動跳出，可依畫面提示複製登入連結。

Windows 若安裝後找不到 Git Bash，先用 where.exe bash 找出 bash.exe，再寫入 ~/.claude/settings.json 的 CLAUDE_CODE_GIT_BASH_PATH。

安裝重點

最容易卡住的地方
  • Git PATH
    Git for Windows 安裝時要選可供 command line 與 3rd-party software 使用。
  • PATH 警告
    PowerShell 若提示路徑不存在，可手動把 $HOME\.local\bin 加入使用者 PATH。
  • 更新方式
    原生安裝可自動更新；WinGet / Homebrew 需手動升級。

核心指令速查

只保留最常用的操作

啟動與續聊

claude 啟動互動模式

claude "修復登入 bug" 直接給任務

claude -c 繼續上次對話

claude -p "摘要這個檔案" 輸出後退出

Slash Commands

/help 查看指令

/clear 重開對話

/model 切換模型

/plan 先規劃再執行

/diff 看未提交修改

特殊語法

! npm test 直接執行 shell

@src/main.ts 指定檔案給 Claude 閱讀

git diff | claude -p "幫我 review" 用管線做快速審查

權限模式

default：初學者安全模式

acceptEdits：自動接受常見編輯

plan：先提案再動手

auto：長任務、自動化工作流

建議工作流程

最適合新手的使用方式

先讀後改

先請 Claude 解釋架構、列出風險或擬定修改計畫，再進入編輯。

先用 plan

不熟專案時，先用 /plan 或 plan 權限模式，避免一次改太多。

搭配 Git

用 /diff 或 git diff 檢查修改內容，再決定是否提交。

FAQ 與實務提醒

精簡版常見問題

Claude Code 要付費嗎？

要。需使用 Claude Pro 或 Max，免費方案無法使用 Claude Code。

可以存取我整台電腦嗎？

預設只在你的工作目錄內操作，且受權限模式與保護路徑限制，可用 /permissions 再細調。

哪些功能值得進一步學？

專案記憶檔 CLAUDE.md、MCP 外部工具整合、IDE 擴充套件、以及 /compact 的 context 管理。

什麼時候用哪個模型？

Opus 適合複雜專案，Sonnet 適合日常開發，Haiku 適合快速查詢。
