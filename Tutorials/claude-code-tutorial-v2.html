什麼是 Claude Code？

Claude Code 是 Anthropic 推出的官方 AI 程式設計助手，它以命令列工具（CLI）的形式運行在你的終端機中。它可以讀取你的程式碼、編輯檔案、執行終端指令，並與 Git、GitHub 等開發工具深度整合。

無論你是要修復 Bug、開發新功能、重構程式碼或理解現有專案，Claude Code 都能以自然語言對話的方式協助你完成工作。

AI 程式助手

以自然語言下達指令，Claude 會理解你的需求並自動完成程式撰寫、除錯等工作

終端機整合

直接在終端機中運行，無需切換視窗，支援與 VS Code 和 JetBrains 整合

多平台支援

支援 macOS、Windows、Linux，同時提供桌面應用程式、Web 版與 CLI 版本

安全權限控制

多層權限模式讓你完全掌控 Claude 能夠執行的操作範圍

系統需求

在安裝 Claude Code 之前，請確認你的系統符合以下需求：

作業系統最低版本備註
macOS13.0 或更新Intel 與 Apple Silicon 皆支援
Windows10（Build 1809+）需先安裝 Git for Windows
LinuxUbuntu 20.04+ / Debian 10+支援 x64 與 ARM64
WSLWSL 1 或 WSL 2WSL 2 支援沙箱安全功能

先決條件

  • 至少 4 GB RAM
  • 穩定的網路連線
  • Claude Pro 或 Max 訂閱方案（免費方案不含 Claude Code）
  • Windows 使用者：需安裝 Git for Windows
  • 位於 Anthropic 支援的國家/地區
注意 Claude Code 需要付費的 Claude Pro 或 Max 訂閱方案。免費的 Claude.ai 帳號無法使用 Claude Code。

Git for Windows 安裝

Windows 使用者在安裝 Claude Code 之前，必須先安裝 Git for Windows。Claude Code 內部使用 Git Bash 來執行 shell 指令，因此這是必要的先決條件。

重要 如果你使用的是 macOS 或 Linux，可以跳過此章節，直接前往安裝指南。

下載與安裝步驟

1

下載安裝程式

前往 Git 官方網站下載 Windows 版本的安裝程式：

URL https://git-scm.com/downloads/win

網站會自動偵測你的系統架構（64-bit 或 32-bit），點擊下載對應的安裝檔（例如 Git-2.x.x-64-bit.exe）。

2

執行安裝程式

雙擊下載的 .exe 檔案啟動安裝精靈。以下是關鍵安裝選項的建議設定：

3

選擇元件（Select Components）

保留預設勾選項目即可。建議確認以下項目已勾選：

  • Git Bash Here — 在右鍵選單加入 Git Bash 啟動選項
  • Git LFS (Large File Support) — 大型檔案支援
  • Associate .git* configuration files with the default text editor
4

選擇預設編輯器（Default Editor）

選擇你慣用的文字編輯器。如果不確定，可以選擇 Use Visual Studio Code as Git's default editor（如果已安裝 VS Code）或保留預設的 Vim。

5

調整 PATH 環境變數（Adjusting your PATH）

建議選擇：Git from the command line and also from 3rd-party software（推薦選項）。這會將 Git 加入系統 PATH，讓 Claude Code 能找到 Git Bash。

注意 請勿選擇 "Use Git from Git Bash only"，否則 Claude Code 可能無法正確找到 Git。
6

選擇 SSH 執行檔

選擇 Use bundled OpenSSH（預設選項）即可。

7

選擇 HTTPS 傳輸後端

選擇 Use the OpenSSL library（預設選項）即可。

8

換行符號轉換（Line Ending Conversions）

建議選擇 Checkout Windows-style, commit Unix-style line endings（預設選項）。這在跨平台專案中最為安全。

9

終端機模擬器（Terminal Emulator）

建議選擇 Use MinTTY (the default terminal of MSYS2)，以獲得較好的終端體驗。

10

其餘選項保持預設

後續的選項（Extra options、Experimental options 等）保持預設值，一路點擊 Next 直到 Install，等待安裝完成後點擊 Finish。

驗證安裝

安裝完成後，開啟 PowerShell 或 CMD，執行以下指令確認 Git 已正確安裝：

PowerShell git --version

如果顯示類似以下輸出，代表安裝成功：

PowerShell
PS C:\Users\you> git --version
git version 2.47.1.windows.1

同時確認 Git Bash 可以正常啟動：

PowerShell bash --version
重要：先記下 bash.exe 的實際位置！ 在繼續安裝 Claude Code 之前，請執行下面這行指令，查出你電腦上 bash.exe 安裝在哪裡。這個路徑之後設定 Claude Code 的 settings.json 時會用到，先查清楚可以省去很多麻煩。
PowerShell where.exe bash
執行後你會看到類似這樣的路徑（每台電腦可能不同）：
C:\Program Files\Git\bin\bash.exe
或者是：
C:\Users\你的使用者名稱\AppData\Local\Programs\Git\bin\bash.exe

請把這個路徑複製起來備用，後面設定時會直接填進去。
安裝成功 看到版本資訊後，代表 Git for Windows 已正確安裝，你可以繼續進行 Claude Code 的安裝。

疑難排解

Claude Code 找不到 Git Bash

如果 Claude Code 無法找到 Git Bash，不用擔心，只需要告訴它 bash.exe 在哪裡就好。請編輯或建立這個設定檔：C:\Users\你的使用者名稱\.claude\settings.json，加入以下內容：

JSON { "env": { "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe" } }
Git 不一定裝在預設位置！請先確認你的實際路徑 上面的路徑（C:\Program Files\Git）是系統層級安裝的預設位置。但如果你是以「一般使用者」身分安裝 Git（沒有勾選「為所有使用者安裝」），Git 很可能裝在這裡：

C:\Users\你的使用者名稱\AppData\Local\Programs\Git\bin\bash.exe

最可靠的方式：在 PowerShell 執行 where.exe bash，它會直接告訴你正確路徑。把那個路徑填進 settings.json 就對了，不用猜。

以下是兩種常見路徑的對照，依照你的情況填入其中一個：

JSON（系統安裝） { "env": { "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe" } }
JSON（使用者安裝） { "env": { "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Users\\你的使用者名稱\\AppData\\Local\\Programs\\Git\\bin\\bash.exe" } }

記得把 你的使用者名稱 換成你電腦上實際的帳號名稱（例如 F096）。

git 指令顯示「不是內部或外部命令」

這表示 Git 沒有正確加入系統 PATH。解決方法：

  • 方法一：重新安裝 Git，在 "Adjusting your PATH" 步驟選擇 "Git from the command line and also from 3rd-party software"
  • 方法二：手動將 Git 加入 PATH：
    1. 按 Win+R，輸入 sysdm.cpl 按 Enter
    2. 切換到「進階」分頁，點擊「環境變數」
    3. 在「系統變數」中找到 Path，點擊「編輯」
    4. 新增以下兩個路徑：
      C:\Program Files\Git\cmd
      C:\Program Files\Git\bin
    5. 確定儲存後，重新開啟終端機視窗
安裝完成後需要重新啟動嗎？

通常不需要重新啟動電腦。但你需要關閉並重新開啟所有已開啟的終端機視窗（PowerShell、CMD），新的 PATH 環境變數才會生效。

安裝指南

根據你的作業系統，選擇對應的安裝方式：

在終端機中執行以下指令：

bash curl -fsSL https://claude.ai/install.sh | bash
自動更新 透過原生安裝方式安裝的 Claude Code 會自動在背景更新。

以 PowerShell 執行以下指令：

PowerShell irm https://claude.ai/install.ps1 | iex
提示 Windows 使用者必須先安裝 Git for Windows。如果 Claude Code 找不到 Git Bash，請在 ~/.claude/settings.json 中設定路徑。
安裝後出現黃色警告「PATH 不存在」怎麼辦？ 如果安裝完成後，PowerShell 顯示類似這樣的黃色警告訊息：
「WARNING: The path '...' does not exist.」

這表示安裝程式想把 Claude Code 加入系統路徑，但那個資料夾還不存在。不用慌，只要執行下面這行指令手動補上就好：
PowerShell [Environment]::SetEnvironmentVariable("Path", $env:Path + ";$HOME\.local\bin", "User")
執行完畢後，關閉 PowerShell 再重新開啟，然後執行 claude --version 確認安裝成功。

在 CMD 中執行以下指令：

CMD curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

使用 WinGet 套件管理員安裝：

PowerShell winget install Anthropic.ClaudeCode
注意 透過 WinGet 安裝的版本不會自動更新。請定期執行 winget upgrade Anthropic.ClaudeCode 來手動更新。

使用 Homebrew 安裝：

bash brew install --cask claude-code
注意 透過 Homebrew 安裝的版本不會自動更新。請定期執行 brew upgrade claude-code 來手動更新。

權限模式（Permission Modes）

權限模式決定 Claude 在執行操作前是否需要你的確認。在會話中按 Shift + Tab 可以切換模式。

default

預設模式。僅自動執行讀取操作，編輯和指令都需確認。適合初學者。

acceptEdits

自動允許檔案編輯與常見檔案操作（mkdir、rm 等），適合信任的開發工作。

plan

計畫模式。Claude 會先提出完整的修改方案，經你批准後才執行。

auto

自動模式。大部分操作無需確認，背景安全檢查仍會運作。適合長時間任務。

啟動時指定模式 claude --permission-mode plan — 以計畫模式啟動
claude --permission-mode acceptEdits — 以自動接受編輯模式啟動

進階功能

當你熟悉基本操作後，可以進一步探索以下進階功能：

CLAUDE.md — 專案記憶檔案

CLAUDE.md 是一個 Markdown 檔案，讓 Claude 在每次啟動時自動讀取你的專案指引。你可以在其中定義程式碼規範、建置指令、架構說明等。

放置位置：

  • ./CLAUDE.md — 專案層級，透過 Git 與團隊共享
  • ~/.claude/CLAUDE.md — 使用者層級，所有專案通用
  • ./CLAUDE.local.md — 個人專案偏好（加入 .gitignore）

範例內容：

CLAUDE.md # 我的專案 ## 建置指令 - 安裝：npm install - 測試：npm test - 部署：npm run deploy ## 程式碼規範 - 使用 TypeScript 嚴格模式 - 使用 2 格空白縮排 - 所有函式都要加 JSDoc 註解 ## 架構 - 前端：React + TypeScript - 後端：Node.js + Express - 資料庫：PostgreSQL

使用 /init 指令可以讓 Claude 自動分析你的專案並產生一份 CLAUDE.md。

MCP 伺服器 — 外部工具整合

MCP（Model Context Protocol）是一個開放協議，讓 Claude Code 可以連接外部工具和服務，例如 GitHub、Slack、Jira、資料庫等。

設定方式：

在 ~/.claude/settings.json 中加入 MCP 伺服器設定：

JSON { "mcpServers": { "github": { "command": "npx", "args": ["@modelcontextprotocol/server-github"], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "your-token" } } } }

管理指令：

  • claude mcp add — 新增 MCP 伺服器
  • claude mcp list — 列出已設定的伺服器
  • claude mcp remove — 移除伺服器
  • /mcp — 在會話中管理 MCP 連線
IDE 整合 — VS Code 與 JetBrains

VS Code 擴充套件：

  • 開啟 VS Code，按 Ctrl+Shift+X，搜尋 “Claude Code”
  • 安裝後即可在側邊欄使用 Claude 面板
  • 支援 inline diff、@-mention 檔案行號、計畫審核模式

VS Code 快捷鍵：

  • Ctrl+Esc — 在編輯器和 Claude 之間切換焦點
  • Ctrl+Shift+Esc — 開啟新對話分頁
  • Alt+K — 插入 @-mention 含檔案路徑和行號

JetBrains 外掛：

從 JetBrains Marketplace 搜尋 “Claude Code Beta” 安裝。支援 IntelliJ、PyCharm、WebStorm 等所有 JetBrains IDE。

桌面應用程式 & Web 版本

除了終端機 CLI 之外，Claude Code 還有其他使用方式：

桌面應用程式（Desktop App）：

  • 支援 macOS 與 Windows
  • 獨立應用程式，無需終端機或 IDE
  • 視覺化 Diff 檢視、可同時開啟多個工作階段

Web 版（claude.ai/code）：

  • 透過瀏覽器使用，無需安裝
  • 可執行長時間任務（電腦關機也不受影響）
  • 支援雲端排程任務
  • 可在手機上透過 Claude iOS App 使用
Context Window 與 /compact

Claude Code 使用 200,000 Token 的 Context Window（使用 1M 模型時更大）。隨著對話進行和檔案讀取，Context 會逐漸被佔滿。

Context 消耗來源：

  • 系統提示（約 4,200 tokens）
  • CLAUDE.md 檔案內容
  • 對話記錄
  • Claude 讀取的檔案內容
  • 工具輸出結果

/compact 指令：

  • 手動執行：輸入 /compact 壓縮對話
  • 自動觸發：Context 接近上限時會自動壓縮
  • 壓縮後保留：對話摘要、CLAUDE.md 設定、自動記憶

最佳化建議：

  • CLAUDE.md 保持在 200 行以內
  • 使用 /context 查看目前使用量
  • 使用 .claude/rules/ 依路徑載入局部規則
可用模型與切換方式

Claude Code 提供多個 AI 模型，適合不同場景：

模型特點適用場景
opus最強大、最具推理能力複雜任務、多步驟專案
sonnet效能與速度平衡一般程式撰寫工作
haiku最快速、最輕量簡單查詢、快速回應

切換模型的方式：

  • 會話中輸入 /model 選擇模型
  • 啟動時指定：claude --model sonnet
  • 按 Alt+P 快速切換
  • 模型名稱加 [1m] 後綴可啟用 1M Token Context：opus[1m]

常見問題

Claude Code 需要付費嗎？

是的。Claude Code 需要 Claude Pro（每月 $20 USD）或 Claude Max 訂閱方案。免費的 Claude.ai 帳號無法使用 Claude Code。企業用戶可透過 Claude for Teams 或 Enterprise 方案使用。

安裝失敗怎麼辦？

請嘗試以下步驟：

  • 執行 claude doctor 進行診斷
  • 確認系統版本是否符合需求
  • Windows 使用者確認已安裝 Git for Windows
  • 確認網路連線正常
  • 嘗試重新安裝：先移除舊版本再安裝新版本
如何更新 Claude Code？

透過原生安裝方式安裝的 Claude Code 會自動背景更新。如果需要手動更新：

  • 執行 claude update
  • WinGet：winget upgrade Anthropic.ClaudeCode
  • Homebrew：brew upgrade claude-code
Claude Code 支援哪些程式語言？

Claude Code 支援所有主流程式語言，包括但不限於：TypeScript、JavaScript、Python、Go、Rust、Java、C/C++、C#、Ruby、PHP、Swift、Kotlin 等。它透過閱讀和理解你的程式碼來提供協助，不限於特定語言。

Claude Code 會存取我電腦上的所有檔案嗎？

Claude Code 預設只能存取你啟動它時所在的工作目錄中的檔案。它有完整的權限控制系統：

  • 受保護的路徑（如 .git、.env 等）永遠需要確認
  • 你可以使用不同的權限模式控制自動允許的操作
  • 使用 /permissions 自訂允許和拒絕規則
如何在不同的專案之間切換？

Claude Code 會自動適應你啟動時所在的目錄：

  • 先 cd 到你的專案目錄
  • 然後執行 claude 啟動
  • Claude 會自動讀取該目錄下的 CLAUDE.md
  • 你也可以使用 /add-dir 在會話中新增額外目錄
如何登出或切換帳號？

在 Claude Code 的會話中輸入 /logout 即可登出。再次啟動 claude 時會重新要求登入。
